# 와드로그 화면별 UX 명세

이 문서는 와드로그 주요 화면의 UX 목표, 정보 구조, 컨트롤 배치, 상태 문구, 테스트 기준을 정의한다. 구현 코드나 Navigation 계약을 확정하지 않는다.

## 1. Home

### 화면 목적

오늘 기록 상태를 확인하고 WOD 추가, 최근 기록 확인, 질문지 생성 흐름으로 빠르게 진입한다.

### 주요 사용자 행동

- 오늘의 WOD 추가
- 오늘 기록 확인
- 최근 3회 기록 확인
- ChatGPT 질문지 생성 흐름 진입

### 정보 구조

- 오늘 날짜와 기록 여부
- 오늘 WOD 카드 또는 빈 상태
- 빠른 추가 액션
- 최근 3회 WOD 요약
- 질문지 생성 바로가기

### 주요 액션

- WOD 추가
- 오늘 기록 보기
- 질문지 생성
- 최근 기록 상세 보기

### 권장 컨트롤 배치

- 화면 상단: 오늘 날짜와 상태
- 본문 첫 영역: 오늘 WOD 카드 또는 빈 상태
- 하단 엄지 접근 영역: WOD 추가 버튼
- 최근 기록은 카드 리스트로 배치

### 빈 상태 문구

- 오늘의 WOD를 추가하세요
- 기록이 아직 없습니다

### 오류 상태 문구

- 기록을 불러오지 못했습니다
- 다시 시도해 주세요

### 성공/완료 피드백

- 저장되었습니다
- 오늘 기록이 추가되었습니다

### 위험 액션 처리 방식

Home에서는 위험 액션을 직접 노출하지 않는다. 삭제나 초기화는 상세 또는 설정 화면에서 확인 다이얼로그를 통해 처리한다.

### 테스트 기준

- 오늘 기록이 없을 때 추가 흐름이 명확하다.
- 오늘 기록이 있을 때 상세 진입이 가능하다.
- 최근 기록 카드가 날짜, 제목, 결과를 빠르게 보여준다.

## 2. Calendar

### 화면 목적

월간 기록 현황을 확인하고 특정 날짜의 WOD로 이동한다.

### 주요 사용자 행동

- 월 이동
- 날짜 선택
- 기록 있는 날짜 확인
- 기록 없는 날짜에 WOD 추가

### 정보 구조

- 월 표시와 이전/다음 월 이동
- 월간 캘린더
- 선택 날짜 요약
- 기록 또는 추가 액션

### 주요 액션

- 이전/다음 월
- 날짜 선택
- WOD 추가
- 상세 보기

### 권장 컨트롤 배치

- 상단: 월과 이동 버튼
- 중앙: 캘린더 그리드
- 하단: 선택 날짜의 기록 카드 또는 추가 버튼

### 빈 상태 문구

- 이 날짜에는 기록이 없습니다
- 새 WOD를 추가하세요

### 오류 상태 문구

- 캘린더 기록을 불러오지 못했습니다

### 성공/완료 피드백

- 선택한 날짜에 기록이 추가되었습니다

### 위험 액션 처리 방식

Calendar에서는 삭제를 직접 제공하지 않는다. 상세 화면에서 확인 후 삭제한다.

### 테스트 기준

- 기록 있는 날짜와 없는 날짜의 시각 차이가 명확하다.
- 선택 날짜가 색상만이 아니라 형태나 강조로도 구분된다.
- 작은 화면에서도 날짜 숫자와 상태 표시가 겹치지 않는다.

## 3. WOD Edit

### 화면 목적

사용자가 WOD 원문과 구조화된 Movement 정보를 부담 없이 입력하거나 수정한다.

### 주요 사용자 행동

- 날짜 선택
- 제목 입력
- WOD Type 선택
- WOD 원문 입력
- 섹션 및 Movement 추가
- Movement별 수치 입력
- 저장

### 정보 구조

- 기본 정보: 날짜, 제목, WOD Type
- WOD 원문
- 섹션별 Movement
- 메모
- 저장 액션

### 주요 액션

- 저장
- 섹션 추가
- Movement 추가
- Movement 삭제
- WOD 삭제

### 권장 컨트롤 배치

- 상단: 날짜, 제목, WOD Type
- 중간: WOD 원문과 구조화 입력을 분리
- Movement 입력은 접거나 펼칠 수 있는 카드 구조 권장
- 하단: 저장 버튼 고정 또는 명확한 하단 액션 영역

### 빈 상태 문구

- Movement를 추가하세요
- WOD 원문을 입력하거나 필요한 항목만 추가하세요

### 오류 상태 문구

- WOD 제목을 입력해 주세요
- 저장할 WOD 유형을 선택하세요
- 입력값을 확인해 주세요

### 성공/완료 피드백

- WOD가 저장되었습니다
- 수정되었습니다

### 위험 액션 처리 방식

- WOD 삭제는 확인 다이얼로그를 사용한다.
- Movement 삭제는 되돌리기 가능 여부가 없다면 확인 또는 스낵바 undo를 검토한다.

### 테스트 기준

- 운동 직후 한 손으로 제목, 유형, 원문, 저장까지 빠르게 진행할 수 있다.
- 모르는 수치 값을 비워둬도 저장 흐름이 막히지 않는다.
- 긴 WOD 원문이 입력 필드 밖으로 겹치지 않는다.

## 4. Result Edit

### 화면 목적

WOD 결과를 Score Type에 맞게 빠르게 저장한다.

### 주요 사용자 행동

- Score Type 선택
- 기록값 입력
- Rx/Scaled/Custom/Unknown 선택
- RPE와 컨디션 선택
- 메모 입력
- 저장

### 정보 구조

- 연결된 WOD 요약
- Score Type 선택
- Score Type별 입력 필드
- Rx 상태
- RPE, 컨디션
- 메모

### 주요 액션

- 저장
- 결과 수정
- 결과 삭제

### 권장 컨트롤 배치

- 상단: 연결된 WOD 제목과 날짜
- 첫 입력 그룹: Score Type Chip
- 선택한 Score Type에 필요한 필드만 표시
- 하단: 저장 버튼

### 빈 상태 문구

- 저장할 결과 유형을 선택하세요

### 오류 상태 문구

- 기록값을 입력해 주세요
- 입력한 시간을 확인해 주세요

### 성공/완료 피드백

- 결과가 저장되었습니다

### 위험 액션 처리 방식

결과 삭제는 확인 다이얼로그를 사용한다.

### 테스트 기준

- Time 선택 시 시간 입력만 중심으로 보인다.
- AMRAP 선택 시 rounds와 reps 입력이 명확하다.
- Rx 상태 선택 여부가 결과 카드에 반영된다.

## 5. WOD Detail

### 화면 목적

하나의 WOD 기록 전체를 확인하고 편집, 결과 입력, 질문지 생성, GPT 답변 보기로 이동한다.

### 주요 사용자 행동

- WOD 원문 확인
- 구조화된 Movement 확인
- 결과 확인
- 메모 확인
- 질문지 생성
- GPT 답변 저장/보기
- 편집 또는 삭제

### 정보 구조

- 날짜, 제목, WOD Type
- WOD 원문
- 섹션별 Movement
- 결과
- 메모
- GPT 질문지/답변 연결 상태
- 관리 액션

### 주요 액션

- 편집
- 결과 입력/수정
- 질문지 생성
- GPT 답변 저장/보기
- 삭제

### 권장 컨트롤 배치

- 상단: 제목과 주요 상태
- 본문: 원문, 구조화 정보, 결과, 메모 순서
- 하단 또는 액션 영역: 질문지 생성, 결과 수정, 편집
- 삭제는 일반 액션과 분리

### 빈 상태 문구

- 결과가 아직 없습니다
- 저장된 GPT 답변이 없습니다

### 오류 상태 문구

- WOD 정보를 불러오지 못했습니다

### 성공/완료 피드백

- 삭제되었습니다
- 변경사항이 저장되었습니다

### 위험 액션 처리 방식

WOD 삭제는 연결된 section, movement, result, prompt, report 삭제 가능성을 명시하는 확인 다이얼로그를 사용한다.

### 테스트 기준

- WOD 원문, 구조화 정보, 결과가 서로 구분된다.
- GPT 답변 저장 여부를 즉시 알 수 있다.
- 삭제 액션이 실수로 눌리기 어렵다.

## 6. Compare

### 화면 목적

최근 3회 WOD를 정량 요약 중심으로 비교한다.

### 주요 사용자 행동

- 최근 3회 기록 확인
- WOD별 수치 요약 확인
- 동일 WOD 반복 여부 확인
- 데이터 부족 상태 이해

### 정보 구조

- 최근 3회 WOD 카드
- WOD Type, Rx, RPE 요약
- 총 reps, load volume, distance, calories
- 카테고리별 movement 수
- 앱 내부 계산 요약

### 주요 액션

- WOD 상세 보기
- 질문지 생성 흐름 진입

### 권장 컨트롤 배치

- 상단: 비교 기준 안내
- 중앙: 전전 WOD / 전 WOD / 현재 WOD 카드
- 하단: MetricCard와 ComparisonRow

### 빈 상태 문구

- 비교할 기록이 아직 부족합니다
- 기록이 쌓이면 최근 3회를 비교할 수 있습니다

### 오류 상태 문구

- 비교 데이터를 불러오지 못했습니다

### 성공/완료 피드백

별도 성공 피드백보다 데이터 갱신 상태를 자연스럽게 반영한다.

### 위험 액션 처리 방식

Compare에서는 위험 액션을 제공하지 않는다.

### 테스트 기준

- 서로 다른 WOD를 단순 순위로 표현하지 않는다.
- 동일 WOD 반복으로 판단 가능한 경우에만 직접 비교 표현을 사용한다.
- 데이터가 3개 미만이어도 가능한 항목을 보여준다.

## 7. Prompt

### 화면 목적

ChatGPT에 붙여넣을 질문지를 읽고 복사한다.

### 주요 사용자 행동

- 질문지 미리보기
- 질문지 복사
- 선택적으로 ChatGPT 앱/웹으로 이동

### 정보 구조

- 연결된 WOD 정보
- 질문지 미리보기
- 복사 액션
- 저장 여부 또는 생성 시점 정보

### 주요 액션

- 복사
- 질문지 저장
- ChatGPT 열기

### 권장 컨트롤 배치

- 상단: 대상 WOD 요약
- 본문: 섹션화된 질문지 미리보기
- 하단 엄지 접근 영역: 복사 버튼

### 빈 상태 문구

- 생성할 질문지 데이터가 부족합니다

### 오류 상태 문구

- 질문지를 생성하지 못했습니다

### 성공/완료 피드백

- 질문지가 복사되었습니다

### 위험 액션 처리 방식

질문지 삭제 기능이 있다면 확인 다이얼로그를 사용한다.

### 테스트 기준

- 긴 질문지가 스크롤로 읽힌다.
- 복사 버튼이 쉽게 보이고 완료 피드백이 표시된다.
- 화면 어디에도 OpenAI API 직접 호출처럼 보이는 표현이 없다.

## 8. Report

### 화면 목적

사용자가 붙여넣은 GPT 답변을 저장하고 다시 읽는다.

### 주요 사용자 행동

- GPT 답변 붙여넣기
- 사용자 메모 입력
- 저장
- 저장된 답변 읽기

### 정보 구조

- 연결된 WOD 정보
- 질문지 원문 보기 선택
- GPT 답변 입력/표시
- 사용자 메모
- 저장 액션

### 주요 액션

- 저장
- 수정
- 삭제
- 질문지 보기

### 권장 컨트롤 배치

- 상단: 연결된 WOD 요약
- 본문: GPT 답변 입력 또는 읽기 영역
- 긴 답변은 섹션과 문단 간격을 살려 표시
- 하단: 저장 버튼

### 빈 상태 문구

- GPT 답변을 붙여넣어 주세요
- 저장된 GPT 답변이 없습니다

### 오류 상태 문구

- GPT 답변을 입력해 주세요
- 답변을 저장하지 못했습니다

### 성공/완료 피드백

- GPT 답변이 저장되었습니다

### 위험 액션 처리 방식

GPT 답변 삭제는 확인 다이얼로그를 사용한다.

### 테스트 기준

- 긴 GPT 답변이 끊기거나 겹치지 않는다.
- 붙여넣기 영역의 라벨과 안내가 명확하다.
- 저장 후 상세 화면에서 다시 접근 가능하다.

## 9. Profile

### 화면 목적

사용자 신체 정보와 크로스핏 시작일을 저장하고 수정한다.

### 주요 사용자 행동

- 키 입력
- 몸무게 입력
- 크로스핏 시작일 입력
- 저장

### 정보 구조

- 키
- 몸무게
- 크로스핏 시작일
- 자동 계산된 경력 표시

### 주요 액션

- 저장
- 수정

### 권장 컨트롤 배치

- 입력 필드는 단순한 세로 흐름으로 배치
- 저장 버튼은 하단에 명확히 배치
- 경력은 보조 설명 영역에 표시

### 빈 상태 문구

- 프로필 정보를 입력해 주세요

### 오류 상태 문구

- 키를 확인해 주세요
- 몸무게를 확인해 주세요
- 시작일을 확인해 주세요

### 성공/완료 피드백

- 프로필이 저장되었습니다

### 위험 액션 처리 방식

Profile에서는 위험 액션을 제공하지 않는다.

### 테스트 기준

- 입력값 수정 후 저장 상태가 분명하다.
- 시작일 기준 경력 표시가 이해하기 쉽다.

## 10. Lifestyle

### 화면 목적

최근 1주 생활습관 정보를 입력해 질문지 품질을 높인다.

### 주요 사용자 행동

- 주 시작일 선택
- 식습관 요약 입력
- 음주/흡연 여부 선택
- 주 평균 주량/흡연량 입력
- 평균 수면 시간 입력
- 메모 입력

### 정보 구조

- 주간 기준 안내
- 식습관
- 음주
- 흡연
- 수면
- 메모

## 11. Settings

### 화면 목적

앱 정보, 오픈소스 정보, 백업/복구, 데이터 초기화 같은 관리 기능을 제공한다.

### 주요 사용자 행동

- 앱 버전 확인
- GitHub 저장소 확인
- MIT License 확인
- JSON 백업 내보내기
- JSON 복구 가져오기
- 앱 데이터 초기화

### 정보 구조

- 앱 정보
- 오픈소스/라이선스
- 데이터 관리
- 위험 액션

### 주요 액션

- 데이터 내보내기
- 데이터 가져오기
- License 보기
- 앱 데이터 초기화

### 권장 컨트롤 배치

- 일반 정보와 데이터 관리 영역을 분리한다.
- 내보내기와 가져오기는 같은 그룹에 배치한다.
- 초기화는 화면 하단 위험 액션 영역에 분리한다.

### 빈 상태 문구

설정 화면 자체에는 빈 상태를 두지 않는다. 백업 파일 선택 흐름에서는 “백업 파일을 선택하세요”를 사용한다.

### 오류 상태 문구

- 백업 파일을 읽지 못했습니다
- 복구에 실패했습니다

### 성공/완료 피드백

- 백업이 완료되었습니다
- 복구가 완료되었습니다

### 위험 액션 처리 방식

- Import는 기존 데이터를 덮어쓸 수 있음을 확인 다이얼로그로 알린다.
- 앱 데이터 초기화는 “이 작업은 되돌릴 수 없습니다”를 명시한다.

### 테스트 기준

- 백업/복구와 초기화가 시각적으로 구분된다.
- 데이터 손실 가능성이 있는 액션은 항상 확인을 요구한다.
