# 와드로그 디자인 QA 체크리스트

디자인 적용 후 수동 QA에 사용하는 체크리스트다. 각 항목은 실제 기기 또는 에뮬레이터에서 확인한다.

## 1. 공통 체크리스트

- [ ] 앱의 주요 화면이 Material 3 기반으로 일관되게 보인다.
- [ ] 저장, 추가, 복사 버튼이 엄지 접근성이 좋은 위치에 있다.
- [ ] 삭제, 초기화, Import 덮어쓰기 액션이 일반 액션과 시각적으로 구분된다.
- [ ] 모든 위험 액션은 확인 다이얼로그를 거친다.
- [ ] 카드 내부 텍스트와 액션이 겹치지 않는다.
- [ ] 주요 버튼의 터치 영역이 충분하다.
- [ ] 시스템 내비게이션 바와 하단 액션이 겹치지 않는다.
- [ ] 로딩, 빈 상태, 오류 상태가 각 화면에서 어색하지 않다.

## 2. Home

- [ ] 오늘 날짜와 오늘 기록 상태가 첫 화면에서 보인다.
- [ ] 오늘 기록이 없을 때 “오늘의 WOD를 추가하세요” 흐름이 명확하다.
- [ ] 최근 3회 기록 카드가 날짜, 제목, 결과를 빠르게 보여준다.
- [ ] 질문지 생성 진입점이 과도하게 강조되지 않으면서 찾기 쉽다.

## 3. Calendar

- [ ] 기록이 있는 날짜와 없는 날짜의 차이가 명확하다.
- [ ] 선택한 날짜가 색상 외 형태나 강조로도 구분된다.
- [ ] 기록이 없는 날짜를 선택했을 때 새 WOD 작성 흐름이 명확하다.
- [ ] 작은 화면에서 날짜 숫자와 상태 표시가 겹치지 않는다.

## 4. WOD Edit

- [ ] 날짜, 제목, WOD Type 입력 흐름이 빠르게 진행된다.
- [ ] WOD 원문 입력과 구조화 입력이 분리되어 보인다.
- [ ] Movement 추가 버튼이 쉽게 보인다.
- [ ] 사용자가 모르는 수치 값을 비워둬도 화면이 자연스럽다.
- [ ] 저장 버튼이 화면 하단의 엄지 접근 영역에 배치되어 있다.
- [ ] 긴 WOD 원문 입력 시 필드가 깨지지 않는다.

## 5. Result Edit

- [ ] Score Type 선택 후 필요한 필드만 표시된다.
- [ ] Time, rounds + reps, reps, load, distance, calories 입력이 서로 구분된다.
- [ ] Rx, Scaled, Custom, Unknown 상태가 명확히 구분된다.
- [ ] RPE와 컨디션 입력이 과하게 복잡하지 않다.
- [ ] 저장할 결과 유형을 선택하지 않았을 때 안내 문구가 보인다.

## 6. WOD Detail

- [ ] WOD 원문, 구조화 정보, 결과, 메모가 명확히 분리된다.
- [ ] 결과가 없을 때 “결과가 아직 없습니다” 문구가 보인다.
- [ ] GPT 질문지 생성과 GPT 답변 보기 상태가 구분된다.
- [ ] 편집, 결과 입력, 질문지 생성, 삭제 액션의 우선순위가 명확하다.
- [ ] 삭제 버튼은 실수로 누르기 어려운 위치와 스타일이다.

## 7. Compare

- [ ] 최근 3회 WOD가 전전 WOD / 전 WOD / 현재 WOD 흐름으로 보인다.
- [ ] 서로 다른 WOD를 좋다/나쁘다 또는 순위로 단정하지 않는다.
- [ ] reps, load volume, distance, calories 등 수치에 단위가 표시된다.
- [ ] 데이터가 3개 미만일 때 가능한 항목만 보여주고 안내 문구가 보인다.
- [ ] 동일 WOD 반복 여부가 있을 때만 직접 비교 강조를 사용한다.

## 8. Prompt

- [ ] 질문지 미리보기가 긴 텍스트 읽기에 적합하다.
- [ ] 복사 버튼이 하단 엄지 접근 영역에 있다.
- [ ] 복사 후 “질문지가 복사되었습니다” 피드백이 표시된다.
- [ ] OpenAI API 직접 호출처럼 보이는 문구나 동작이 없다.
- [ ] 데이터가 부족한 경우 가능한 항목만 포함했다는 안내가 보인다.

## 9. Report

- [ ] “GPT 답변을 붙여넣어 주세요” 문구가 명확하다.
- [ ] 긴 GPT 답변을 붙여넣어도 입력 영역과 저장 버튼이 겹치지 않는다.
- [ ] 저장된 GPT 답변이 문단 단위로 읽기 쉽다.
- [ ] 사용자 메모와 GPT 답변이 구분된다.
- [ ] GPT 답변 삭제 시 확인 다이얼로그가 표시된다.

## 10. Profile

- [ ] 키, 몸무게, 크로스핏 시작일 입력 필드가 명확하다.
- [ ] 저장 후 “프로필이 저장되었습니다” 피드백이 표시된다.
- [ ] 크로스핏 경력 표시가 이해하기 쉽다.
- [ ] 값 오류 시 사용자를 탓하지 않는 안내 문구가 보인다.

## 11. Lifestyle

- [ ] 주 시작일이 화면 상단에서 명확하다.
- [ ] 주간 단위 입력임을 사용자가 이해할 수 있다.
- [ ] 음주/흡연 여부에 따라 세부 입력 표시가 자연스럽다.
- [ ] 평균 수면시간이 선택 입력처럼 보인다.
- [ ] 저장 후 “생활습관이 저장되었습니다” 피드백이 표시된다.

## 12. Settings

- [ ] 앱 이름, 버전, GitHub 저장소, License 정보가 구분되어 있다.
- [ ] 데이터 내보내기와 가져오기가 같은 데이터 관리 그룹에 있다.
- [ ] 앱 데이터 초기화가 위험 액션 영역에 분리되어 있다.
- [ ] Import 전 현재 데이터 변경 가능성을 알리는 다이얼로그가 보인다.
- [ ] 초기화 전 “이 작업은 되돌릴 수 없습니다” 문구가 보인다.

## 13. 한국어 문구 체크

- [ ] 안내 문구, 버튼, 오류, 빈 상태, 다이얼로그가 한국어로 작성되어 있다.
- [ ] 문구가 짧고 명확하다.
- [ ] 사용자를 탓하는 표현이 없다.
- [ ] 기술 용어가 불필요하게 노출되지 않는다.
- [ ] 저장, 복사, 추가, 삭제 문구가 서로 혼동되지 않는다.

## 14. CrossFit 용어 표기 체크

- [ ] For Time, AMRAP, EMOM, RFT, Chipper가 영어로 표기된다.
- [ ] Rx, Scaled, Custom, RPE가 영어로 표기된다.
- [ ] Thruster, Pull-up, Deadlift, Row 같은 운동명이 영어 표기를 유지한다.
- [ ] 같은 용어가 화면마다 일관되게 표기된다.
- [ ] 필요한 경우 영어 용어 뒤에 짧은 한국어 설명이 붙는다.

## 15. 접근성 체크

- [ ] 주요 터치 영역이 48dp 이상이다.
- [ ] 아이콘 버튼에 의미 있는 접근성 라벨이 있다.
- [ ] Chip 선택 상태가 색상만으로 전달되지 않는다.
- [ ] TalkBack에서 화면 제목, 입력 라벨, 오류 문구를 이해할 수 있다.
- [ ] 글자 크기 확대 시 주요 버튼 문구가 잘리지 않는다.

## 16. 다크 테마 가독성 체크

- [ ] Background와 Surface의 경계가 명확하다.
- [ ] 본문, 보조 설명, 비활성 텍스트 대비가 충분하다.
- [ ] Primary 버튼이 다크 배경에서 선명하다.
- [ ] Error, Warning, Success 색상이 서로 혼동되지 않는다.

## 17. 라이트 테마 가독성 체크

- [ ] Surface와 Background가 구분된다.
- [ ] Outline이 너무 진하거나 너무 흐리지 않다.
- [ ] Primary 버튼과 텍스트 대비가 충분하다.
- [ ] 다크 테마 전용처럼 보이는 색 조합이 없다.

## 18. 작은 화면 대응 체크

- [ ] 360dp 폭 수준의 작은 화면에서 주요 버튼이 잘리지 않는다.
- [ ] BottomBar 라벨이 겹치지 않는다.
- [ ] Calendar 날짜 그리드가 깨지지 않는다.
- [ ] 입력 필드와 하단 저장 버튼이 키보드와 겹치지 않는다.

## 19. 긴 텍스트 대응 체크

- [ ] 긴 WOD 제목이 카드 레이아웃을 깨지 않는다.
- [ ] 긴 WOD 원문이 스크롤 또는 다중 행으로 안정적으로 표시된다.
- [ ] 긴 GPT 답변이 저장/조회 화면에서 겹치지 않는다.
- [ ] 긴 한국어 오류 문구가 버튼이나 입력 필드를 밀어내지 않는다.

## 20. 빈 상태 체크

- [ ] 첫 실행 Home에서 다음 행동이 보인다.
- [ ] Calendar 기록 없음 상태에서 WOD 추가 흐름이 보인다.
- [ ] Compare 기록 부족 상태가 자연스럽다.
- [ ] Report 답변 없음 상태가 붙여넣기 행동을 안내한다.

## 21. 오류 상태 체크

- [ ] 저장 실패 시 사용자에게 재시도 가능성을 안내한다.
- [ ] 입력 오류가 해당 필드 가까이에 표시된다.
- [ ] 백업 파일 읽기 실패 시 원인을 과도하게 기술적으로 쓰지 않는다.
- [ ] 오류 상태에서도 화면 주요 액션이 겹치지 않는다.

## 22. 삭제/초기화 확인 다이얼로그 체크

- [ ] WOD 삭제 전 연결 데이터 삭제 가능성을 안내한다.
- [ ] GPT 답변 삭제 전 확인을 요구한다.
- [ ] 앱 데이터 초기화 전 되돌릴 수 없음을 알린다.
- [ ] 위험 실행 버튼 문구가 “확인”이 아니라 “삭제”, “초기화”처럼 명확하다.

## 23. 백업/복구 흐름 체크

- [ ] 데이터 내보내기 완료 피드백이 표시된다.
- [ ] 데이터 가져오기 전 백업 파일 선택 흐름이 명확하다.
- [ ] Import 전 현재 데이터가 바뀔 수 있음을 안내한다.
- [ ] 복구 완료 후 화면 데이터가 갱신된다.
- [ ] 복구 실패 시 현재 데이터를 잃은 것처럼 오해하지 않게 안내한다.
