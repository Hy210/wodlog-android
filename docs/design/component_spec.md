# 와드로그 공통 컴포넌트 명세

이 문서는 와드로그 UI 고도화에 사용할 공통 컴포넌트의 디자인/UX 기준을 정의한다. 실제 Kotlin/Compose API를 확정하지 않으며, Material 3 기본 컴포넌트를 우선 활용하는 방향만 제시한다.

## 1. WodLogScaffold

- 목적: 화면 공통 구조, 안전 영역, TopBar/BottomBar/Content/Action 영역을 일관되게 제공한다.
- 사용 위치: 모든 주요 화면.
- 표시 정보: 화면 콘텐츠, 상단 제목, 하단 네비게이션, 주요 액션 영역.
- 스타일 원칙: 배경과 Surface 계층을 명확히 하고 시스템 내비게이션과 겹치지 않게 한다.
- 상태: 기본, 로딩, 오류, 빈 상태를 화면 단위로 수용한다.
- 접근성 고려사항: 화면 제목이 TalkBack에서 먼저 읽히도록 한다.
- 금지 사항: Scaffold 내부에서 기능 로직을 직접 처리하지 않는다.

## 2. WodLogTopBar

- 목적: 현재 화면 제목과 뒤로가기, 보조 액션을 제공한다.
- 사용 위치: 상세, 편집, 설정 하위 화면.
- 표시 정보: 화면 제목, 뒤로가기, 편집/삭제 같은 보조 액션.
- 스타일 원칙: 제목은 짧게, 액션은 아이콘 또는 명확한 텍스트로 제공한다.
- 상태: 기본, 스크롤 후 강조, 액션 비활성.
- 접근성 고려사항: 아이콘 버튼에 명확한 라벨을 제공한다.
- 금지 사항: 삭제 같은 위험 액션을 눈에 띄지 않는 아이콘 하나로만 숨기지 않는다.

## 3. WodLogBottomBar

- 목적: 주요 탭 이동을 제공한다.
- 사용 위치: Home, Calendar, Compare, Settings 같은 최상위 화면.
- 표시 정보: 탭 아이콘과 한국어 라벨.
- 스타일 원칙: 현재 선택 탭이 명확해야 한다.
- 상태: 기본, 선택됨, 비활성.
- 접근성 고려사항: 선택됨 상태가 TalkBack으로 전달되어야 한다.
- 금지 사항: 상세/편집 화면에서 저장 버튼보다 네비게이션이 더 강조되지 않게 한다.

## 4. WodLogCard

- 목적: 하나의 정보 묶음을 카드로 표현한다.
- 사용 위치: Home 최근 기록, WOD Detail, Compare, Settings 그룹.
- 표시 정보: 제목, 보조 정보, 본문, 선택 액션.
- 스타일 원칙: 8dp 안팎의 절제된 radius, 적절한 Surface와 Outline 사용.
- 상태: 기본, 선택됨, 비활성, 오류.
- 접근성 고려사항: 카드 전체가 클릭 가능하면 클릭 목적을 라벨로 제공한다.
- 금지 사항: 카드 안에 큰 카드 컨테이너를 중첩하지 않는다.

## 5. SectionHeader

- 목적: 화면 내 정보 그룹의 시작을 알린다.
- 사용 위치: WOD 원문, Movement, 결과, 메모, GPT 답변 섹션.
- 표시 정보: 섹션 제목, 선택적 보조 설명 또는 액션.
- 스타일 원칙: 카드 제목보다 상위 구조를 나타내되 과도하게 크지 않게 한다.
- 상태: 기본, 보조 액션 포함, 비활성.
- 접근성 고려사항: heading 역할을 고려한다.
- 금지 사항: 긴 설명을 헤더에 모두 넣지 않는다.

## 6. PrimaryActionButton

- 목적: 저장, 추가, 복사 같은 핵심 액션을 수행한다.
- 사용 위치: WOD Edit, Result Edit, Prompt, Report, Profile.
- 표시 정보: 짧은 한국어 동사. 예: 저장, 추가, 복사.
- 스타일 원칙: 화면 하단 엄지 접근 영역에 배치하고 충분한 높이를 확보한다.
- 상태: 기본, 비활성, 로딩.
- 접근성 고려사항: 48dp 이상 터치 영역을 확보한다.
- 금지 사항: 삭제나 초기화 같은 위험 액션에 사용하지 않는다.

## 7. SecondaryActionButton

- 목적: 보조 이동이나 선택 액션을 제공한다.
- 사용 위치: 상세 보기, 질문지 보기, License 보기, ChatGPT 열기.
- 표시 정보: 짧은 한국어 문구.
- 스타일 원칙: Primary보다 낮은 시각 강조를 사용한다.
- 상태: 기본, 비활성, 로딩.
- 접근성 고려사항: Primary와 목적이 구분되도록 라벨을 작성한다.
- 금지 사항: Primary와 같은 위치와 강조로 경쟁시키지 않는다.

## 8. DangerActionButton

- 목적: 삭제, 초기화, Import 덮어쓰기 등 위험 액션을 수행한다.
- 사용 위치: WOD Detail, Report, Settings.
- 표시 정보: 삭제, 초기화, 덮어쓰기 같은 명확한 문구.
- 스타일 원칙: Error 색상 또는 위험 영역으로 일반 액션과 구분한다.
- 상태: 기본, 비활성, 로딩.
- 접근성 고려사항: 위험성을 라벨 또는 확인 다이얼로그에서 명확히 설명한다.
- 금지 사항: 확인 다이얼로그 없이 즉시 실행하지 않는다.

## 9. WodLogTextField

- 목적: 한국어 문구, WOD 원문, 수치, 메모, GPT 답변을 입력한다.
- 사용 위치: 모든 입력 화면.
- 표시 정보: 라벨, 값, 단위, 보조 설명, 오류 문구.
- 스타일 원칙: 필수/선택 상태와 오류 상태를 명확히 표시한다.
- 상태: 기본, 포커스, 비활성, 오류, 로딩 불필요.
- 접근성 고려사항: 라벨과 오류 문구가 입력 필드와 연결되어야 한다.
- 금지 사항: 기술적인 DB 필드명을 그대로 노출하지 않는다.

## 10. WodLogDropdown

- 목적: 날짜, 카테고리, 컨디션처럼 선택지가 명확한 항목을 선택한다.
- 사용 위치: Profile, Lifestyle, Movement category, Condition.
- 표시 정보: 현재 선택값, 선택 가능한 값, 보조 설명.
- 스타일 원칙: 선택값이 없으면 한국어 placeholder를 제공한다.
- 상태: 기본, 선택됨, 비활성, 오류.
- 접근성 고려사항: 펼침 가능 상태와 현재 선택값을 읽을 수 있어야 한다.
- 금지 사항: WOD Type처럼 빠른 선택이 중요한 항목을 무조건 Dropdown으로 숨기지 않는다.

## 11. WodTypeChip

- 목적: WOD 유형을 빠르게 선택한다.
- 사용 위치: WOD Edit, WOD Detail 요약.
- 표시 정보: For Time, AMRAP, EMOM, RFT, Strength, Skill, Chipper 등.
- 스타일 원칙: CrossFit 용어는 영어로 유지한다.
- 상태: 기본, 선택됨, 비활성, 오류.
- 접근성 고려사항: 선택됨 상태가 색상 외 방법으로도 전달되어야 한다.
- 금지 사항: 용어를 임의로 한국어 번역하지 않는다.

## 12. ScoreTypeChip

- 목적: 결과 기록 유형을 빠르게 선택한다.
- 사용 위치: Result Edit.
- 표시 정보: Time, Rounds + reps, Reps, Load, Distance, Calories, Other.
- 스타일 원칙: 선택 즉시 필요한 입력 필드만 노출되는 흐름을 지원한다.
- 상태: 기본, 선택됨, 비활성, 오류.
- 접근성 고려사항: 선택 후 바뀐 입력 영역을 사용자가 이해할 수 있게 한다.
- 금지 사항: 선택하지 않은 Score Type의 입력 필드를 과도하게 한 화면에 노출하지 않는다.

## 13. RxScaledChip

- 목적: Rx, Scaled, Custom, Unknown 상태를 표시하거나 선택한다.
- 사용 위치: Result Edit, Detail, Compare.
- 표시 정보: Rx, Scaled, Custom, Unknown.
- 스타일 원칙: Rx는 성취 강조, Scaled/Custom은 중립적 정보로 표현한다.
- 상태: 기본, 선택됨, 비활성, 오류.
- 접근성 고려사항: 난이도 상태를 명확히 읽어준다.
- 금지 사항: Scaled를 부정적으로 보이게 표현하지 않는다.

## 14. EmptyState

- 목적: 데이터가 없을 때 다음 행동을 안내한다.
- 사용 위치: Home, Calendar 선택 날짜, Compare, Prompt, Report.
- 표시 정보: 짧은 제목, 보조 설명, 선택적 액션 버튼.
- 스타일 원칙: 가볍고 명확하게, 장식보다 다음 행동 중심.
- 상태: 기본, 액션 포함, 데이터 부족.
- 접근성 고려사항: 빈 상태와 가능한 다음 행동이 함께 읽혀야 한다.
- 금지 사항: 사용자를 탓하는 문구를 사용하지 않는다.

## 15. ConfirmDialog

- 목적: 위험 액션 실행 전 확인을 받는다.
- 사용 위치: 삭제, 초기화, Import 덮어쓰기.
- 표시 정보: 제목, 짧은 설명, 취소, 실행.
- 스타일 원칙: 위험 실행 버튼은 Error 성격으로 구분한다.
- 상태: 기본, 로딩, 오류.
- 접근성 고려사항: 다이얼로그 제목과 되돌릴 수 없는 작업 여부를 읽을 수 있어야 한다.
- 금지 사항: “확인”처럼 모호한 실행 버튼만 사용하지 않는다.

## 16. MetricCard

- 목적: 수치 요약을 빠르게 보여준다.
- 사용 위치: Home, Compare, Detail 결과 영역.
- 표시 정보: 값, 단위, 라벨, 보조 설명.
- 스타일 원칙: 숫자와 단위의 위계를 명확히 하고 정렬을 일관되게 한다.
- 상태: 기본, 데이터 없음, 오류.
- 접근성 고려사항: 값과 단위를 함께 읽도록 한다.
- 금지 사항: 단위 없는 숫자만 표시하지 않는다.

## 17. ComparisonRow

- 목적: 최근 3회 WOD의 동일 지표를 행 단위로 비교한다.
- 사용 위치: Compare.
- 표시 정보: 지표명, 전전 WOD 값, 전 WOD 값, 현재 WOD 값, 단위.
- 스타일 원칙: 서로 다른 WOD의 우열 표현보다 값의 병렬 표시를 우선한다.
- 상태: 기본, 일부 데이터 없음, 동일 WOD 강조.
- 접근성 고려사항: 컬럼 순서가 TalkBack에서 이해 가능해야 한다.
- 금지 사항: 서로 다른 WOD를 순위나 승패로 표현하지 않는다.

## 18. PromptPreviewCard

- 목적: ChatGPT 질문지를 읽고 복사하기 쉽게 미리보기로 제공한다.
- 사용 위치: Prompt.
- 표시 정보: 질문지 텍스트, 생성 시점, 연결된 WOD 정보.
- 스타일 원칙: 긴 텍스트는 문단 간격과 스크롤을 고려한다.
- 상태: 기본, 로딩, 오류, 데이터 부족.
- 접근성 고려사항: 긴 텍스트 영역과 복사 버튼의 목적이 구분되어야 한다.
- 금지 사항: API 호출 또는 자동 전송처럼 보이는 표현을 사용하지 않는다.

## 19. ReportCard

- 목적: 저장된 GPT 답변을 읽기 좋은 카드로 표시한다.
- 사용 위치: WOD Detail, Report.
- 표시 정보: 답변 본문, 저장일, 사용자 메모, 연결된 질문지 여부.
- 스타일 원칙: 긴 답변은 섹션과 문단이 구분되어야 한다.
- 상태: 기본, 편집 가능, 빈 상태, 오류.
- 접근성 고려사항: 긴 본문 스크롤과 액션 버튼이 겹치지 않게 한다.
- 금지 사항: GPT 답변을 앱 내부 분석 결과처럼 오인시키지 않는다.
