# 네이버카페 WOD 가져오기 설계 명세

작성일: 2026-05-05  
대상 앱: 와드로그 Android  
문서 목적: 다음 버전에서 구현할 **네이버카페 WOD 가져오기** 기능의 제품, UX, 데이터, 보안, 테스트 기준을 정의한다.  
주의: 이 문서는 설계 문서이며 Kotlin/Compose, Gradle, DB, DAO, Repository, ViewModel, Navigation 코드를 수정하지 않는다.

---

## 1. 기능 개요

- 기능명: 네이버카페 WOD 가져오기
- 목표: 사용자가 다니는 CrossFit box의 네이버카페 WOD 게시판 URL에서 WOD 게시글을 가져와 WOD 입력란에 자동 채운다.
- 대상: box별로 서로 다른 네이버카페 게시판 URL
- 방식: 앱 내 WebView 기반 사용자 주도 가져오기
- 홈 화면 진입점: 설정된 카페 URL이 있으면 홈 화면의 기존 **WOD 추가** 버튼 아래에 초록색 **WOD 불러오기** 버튼을 표시한다.

이 기능은 자동 크롤링 기능이 아니라 사용자가 직접 접근한 현재 WebView 페이지에서만 텍스트와 링크 후보를 추출하는 입력 보조 기능이다. 사용자는 저장 전에 반드시 가져온 내용을 확인하고 수정할 수 있어야 한다.

---

## 2. 지원 범위

- 사용자가 직접 등록한 네이버카페 게시판 URL
- WebView에서 사용자가 직접 접근 가능한 게시글 목록
- 사용자가 직접 로그인한 상태에서 볼 수 있는 게시글
- 현재 표시된 페이지에서 게시글 후보 추출
- 선택한 게시글의 본문 텍스트 추출
- WOD 입력 화면의 `rawText` 자동 채움
- 저장 전 사용자 확인 및 수정
- 설정된 카페 URL이 있을 경우 홈 화면에서 **WOD 불러오기** 버튼 제공
- 카페 URL이 여러 개일 경우 카페 소스 선택 후 가져오기 진행

## 4. 사용자 흐름

### A. 카페 URL 등록

1. 사용자는 설정 화면 또는 가져오기 화면에서 카페 소스를 추가한다.
2. `box name`을 입력한다.
3. 네이버카페 게시판 URL을 입력한다.
4. WOD 게시글 제목 키워드를 입력한다.
   - 기본값 예: `WOD`, `와드`, `오늘의 와드`, `Metcon`, `Workout`
5. 필요하면 모바일 URL 우선 사용 옵션을 설정한다.
6. 저장한다.

### B. 홈 화면에서 가져오기 진입

1. 앱은 등록된 `CafeSource` 개수를 확인한다.
2. 사용자가 카페 URL을 하나 이상 등록한 경우 HomeScreen의 **WOD 추가** 버튼 아래에 초록색 **WOD 불러오기** 버튼을 표시한다.
3. 카페 URL이 등록되어 있지 않으면 **WOD 불러오기** 버튼은 숨기거나 비활성화한다.
4. 버튼 클릭 시 동작:
   - 등록된 카페가 1개면 바로 해당 카페 가져오기 화면으로 이동한다.
   - 등록된 카페가 2개 이상이면 카페 선택 화면 또는 BottomSheet를 표시한다.
5. 사용자가 카페 소스를 선택하면 WebView 기반 가져오기 흐름으로 진입한다.

### C. WOD 가져오기 시작

1. 사용자는 오늘 WOD 추가 화면 또는 홈 화면의 **WOD 불러오기** 버튼에서 진입한다.
2. 필요한 경우 등록된 카페 소스를 선택한다.
3. 앱은 선택된 게시판 URL을 WebView로 연다.

### D. 로그인 필요 시

1. 게시판이 로그인을 요구하면 사용자는 WebView 안에서 직접 네이버에 로그인한다.
2. 앱은 네이버 ID/PW를 읽거나 저장하지 않는다.
3. 앱은 자동 로그인을 시도하지 않는다.
4. 로그인 후 사용자는 게시판 목록으로 이동한다.

### E. 게시글 후보 추출

1. 사용자가 **현재 목록에서 WOD 찾기** 버튼을 누른다.
2. 앱은 현재 WebView 페이지에서 텍스트와 링크 후보를 추출한다.
3. 제목, 날짜, 키워드 기반으로 `CafePostCandidate` 목록을 만든다.
4. 후보 목록은 confidence가 높은 순서로 표시하되, confidence는 정답 판정이 아니라 정렬용 점수로 사용한다.

### F. 게시글 선택

1. 사용자는 후보 게시글을 직접 선택한다.
2. 앱은 WebView에서 해당 게시글 URL을 연다.

### G. 본문 가져오기

1. 게시글 페이지가 열린 뒤 사용자가 **본문 가져오기** 버튼을 누른다.
2. 앱은 현재 게시글 페이지의 본문 텍스트를 추출한다.
3. 추출 결과를 `ImportedWodPreviewScreen`에서 미리보기로 표시한다.

### H. WOD 입력란 적용

1. 사용자가 **WOD 입력란에 적용**을 누른다.
2. 앱은 WOD Edit 화면으로 이동한다.
3. `date`, `title`, `rawText`, `sourceType`, `sourceUrl` 정보가 채워진다.
4. 사용자는 내용을 확인하고 수정한 뒤 저장한다.

## 5. 화면 설계

### HomeScreen

- 기존 **WOD 추가** 버튼은 수동 입력 진입점으로 유지한다.
- 설정된 카페 URL이 하나 이상 있으면 **WOD 추가** 버튼 아래에 초록색 **WOD 불러오기** 버튼을 표시한다.
- **WOD 불러오기** 버튼 클릭 시 네이버카페 WOD 가져오기 흐름으로 진입한다.
- 카페 URL이 1개면 바로 가져오기 화면으로 이동한다.
- 카페 URL이 여러 개면 카페 소스 선택 UI를 표시한다.
- 카페 URL이 없으면 버튼을 숨기거나 비활성화한다.
- 버튼 문구는 **WOD 불러오기**를 사용한다.
- 버튼 색상은 초록색 계열로 정의한다.
- 보조 문구를 둘 수 있다.
  - 예: `설정한 네이버카페에서 오늘 WOD를 가져옵니다.`

### CafeSourceSettingsScreen

- 등록된 카페 목록
- 카페 추가
- 카페 수정
- 카페 삭제
- `box name`
- `board URL`
- `title keywords`
- `prefer mobile URL` option

### CafeImportScreen

- 등록된 카페 소스 선택
- URL 직접 입력
- WebView 열기
- 현재 목록에서 WOD 찾기
- 후보 게시글 목록
- 실패 시 수동 입력 안내

### CafeWebViewScreen

- WebView 영역
- 현재 URL 표시
- 뒤로가기
- 새로고침
- 현재 목록에서 WOD 찾기
- 본문 가져오기
- 수동 입력으로 돌아가기

### CafePostCandidateScreen 또는 BottomSheet

- 후보 게시글 제목
- 날짜 텍스트
- URL
- 매칭 키워드
- confidence 표시
- 사용자가 직접 선택

### ImportedWodPreviewScreen

- source URL
- 추출된 본문 미리보기
- WOD 입력란에 적용
- 복사
- 수동 수정

## 6. 데이터 모델 초안

아래 모델은 Kotlin `data class` 또는 Room Entity 관점의 설계 초안이다. 실제 필드명과 타입은 구현 Phase에서 기존 코드 스타일에 맞춰 확정한다.

```kotlin
data class CafeSource(
    val id: Long,
    val boxName: String,
    val boardUrl: String,
    val titleKeywords: List<String>,
    val preferMobileUrl: Boolean,
    val createdAt: Instant,
    val updatedAt: Instant,
)

data class CafePostCandidate(
    val title: String,
    val url: String,
    val dateText: String?,
    val matchedKeyword: String?,
    val confidence: Double,
)

data class ImportedWodText(
    val sourceType: WodSourceType,
    val sourceUrl: String,
    val title: String,
    val importedText: String,
    val importedAt: Instant,
)

enum class WodSourceType {
    MANUAL,
    NAVER_CAFE_WEBVIEW,
    PASTE_TEXT,
    OCR,
}
```

### 기존 Wod 모델 확장 검토

구현 Phase에서 기존 Wod 모델에 다음 필드를 추가할지 검토한다.

- `sourceType`: WOD가 수동 입력인지, 네이버카페 WebView 가져오기인지, 붙여넣기인지, OCR인지 구분한다.
- `sourceUrl`: 네이버카페 게시글 URL을 저장한다. 수동 입력일 경우 null 또는 빈 값이 가능하다.
- `importedAt`: 외부 원문을 가져온 시각을 저장한다. 수동 입력일 경우 null이 가능하다.

확장 장점:

- WOD 기록이 어떤 방식으로 생성되었는지 확인할 수 있다.
- 향후 원문 다시 보기, 출처 표시, 가져오기 실패 분석에 도움이 된다.
- PASTE_TEXT, OCR 같은 후속 입력 방식과 같은 구조로 확장할 수 있다.

주의점:

- Room migration이 필요할 수 있다.
- 기존 WOD 생성 및 수정 흐름에서 기본값 처리가 필요하다.
- 기존 기록은 `MANUAL` 또는 null-safe 기본값으로 해석해야 한다.

### 홈 화면 버튼 표시 데이터 흐름

1. `CafeSourceRepository`가 등록된 `CafeSource` 개수 또는 목록 Flow를 제공한다.
2. `HomeViewModel`은 `hasCafeSources: Boolean` 상태를 제공한다.
3. `HomeScreen`은 `hasCafeSources == true`일 때 **WOD 불러오기** 버튼을 표시한다.
4. `CafeSource`가 로딩 중이면 버튼 영역은 숨김, 비활성화, 또는 로딩 상태 중 하나로 처리한다.
5. 데이터 로딩 실패 시 설정 화면으로 이동 가능한 안내를 표시한다.

## 7. WebView 구현 원칙

- WebView는 사용자가 직접 접근하는 화면으로 사용한다.
- 네이버카페 페이지는 JavaScript 기반 UI와 동적 로딩을 사용할 수 있으므로 JavaScript 활성화가 필요할 수 있다.
- JavaScript 활성화는 WebView 내부 페이지 렌더링과 사용자가 누른 가져오기 동작에 한정한다.
- 사용자가 직접 버튼을 누른 경우에만 현재 페이지의 텍스트를 추출한다.
- 가능한 한 `document.body.innerText`와 링크 목록을 활용하는 단순한 추출 방식부터 시작한다.
- iframe, 모바일/PC 페이지 구조 차이, 네이버카페 구조 변경 가능성을 고려한다.
- WebView 로그인 쿠키를 앱 DB에 저장하지 않는다.
- 네이버 ID/PW를 앱 코드에서 읽거나 저장하거나 전송하지 않는다.
- 자동 로그인, CAPTCHA 우회, 차단 회피를 구현하지 않는다.
- WebView의 현재 URL과 로딩 상태를 사용자에게 명확히 보여준다.

---

## 8. 게시글 후보 추출 규칙

후보 추출은 현재 표시된 목록 페이지에서만 수행한다. 첫 구현은 단순 점수 기반으로 시작한다.

기준:

- 사용자가 등록한 키워드 포함 여부
- 오늘 날짜 포함 여부
- 게시글 목록 상단에 가까운지
- 제목에 `WOD`, `와드`, `Metcon`, `Workout`, `오늘` 등의 단어 포함 여부
- URL이 네이버카페 게시글 링크로 보이는지
- confidence는 절대적 판단이 아니라 후보 정렬용이다.

예시 점수:

| 기준 | 점수 |
|---|---:|
| title contains keyword | +0.4 |
| date looks like today | +0.3 |
| top list item | +0.1 |
| url looks like cafe article | +0.2 |

추출 결과 confidence가 낮더라도 사용자가 직접 후보를 고를 수 있어야 한다. 후보가 없으면 수동 입력 또는 URL 직접 이동을 안내한다.

---

## 9. 본문 추출 규칙

- 현재 WebView 페이지의 전체 텍스트를 추출한다.
- 불필요한 네비게이션, 댓글, 광고, 메뉴 텍스트 제거를 시도한다.
- 너무 공격적인 제거는 하지 않는다.
- 추출 결과는 반드시 미리보기에서 사용자가 확인한다.
- 본문 추출 실패 시 복사/붙여넣기 또는 수동 입력으로 안내한다.
- 본문 텍스트가 지나치게 짧거나 WOD 관련 키워드가 전혀 없으면 실패 또는 낮은 신뢰 상태로 표시한다.

---

## 10. WOD 입력 화면 연동

- `ImportedWodText`를 WOD Edit 화면으로 전달한다.
- `title`은 게시글 제목으로 채운다.
- `rawText`는 추출 본문으로 채운다.
- `sourceType`은 `NAVER_CAFE_WEBVIEW`로 설정한다.
- `sourceUrl`은 게시글 URL로 설정한다.
- `date`는 기본적으로 오늘 날짜로 설정한다.
- 사용자는 날짜를 직접 변경할 수 있다.
- WOD type과 movement 구조화는 사용자가 확인하고 수정한다.
- 자동 구조화 파싱은 MVP 범위가 아니며, 가져온 원문을 빠르게 채워주는 데 집중한다.

## 11. 홈 화면 연동

### 표시 위치

- HomeScreen에서 기존 **WOD 추가** 버튼 아래에 **WOD 불러오기** 버튼을 표시한다.

### 표시 조건

- `CafeSource`가 1개 이상 등록되어 있을 때 표시한다.
- `CafeSource`가 없으면 숨기거나 비활성화한다.

### 버튼 스타일

- 초록색 계열
- Primary 버튼과 구분되는 Success 또는 Import 액션 스타일
- 텍스트: **WOD 불러오기**

### 버튼 동작

- `CafeSource`가 1개면 바로 `CafeImportScreen` 또는 `CafeWebViewScreen`으로 이동한다.
- `CafeSource`가 여러 개면 선택 BottomSheet 또는 CafeSource 선택 화면을 표시한다.

### 보조 안내 문구

- `설정한 네이버카페에서 오늘 WOD를 가져옵니다.`

### 실패 시

- 등록된 카페 URL이 삭제되었거나 유효하지 않으면 설정 화면으로 이동하도록 안내한다.

### UX 원칙

- **WOD 추가**는 수동 입력이다.
- **WOD 불러오기**는 네이버카페 가져오기이다.
- 두 버튼의 역할이 혼동되지 않도록 시각적, 문구적으로 구분한다.
- 운동 직후 사용자가 가장 빠르게 오늘 WOD를 가져올 수 있도록 홈 화면의 주요 CTA로 취급한다.

## 12. 실패 시나리오와 사용자 안내 문구

모든 안내 문구는 한국어로 작성한다. CrossFit 용어와 WOD type은 영어 표기를 유지한다.

| 시나리오 | 안내 문구 |
|---|---|
| URL이 비어 있음 | 게시판 URL을 입력해 주세요. |
| 잘못된 URL | 올바른 네이버카페 게시판 URL을 입력해 주세요. |
| 네이버카페 페이지가 열리지 않음 | 네이버카페 페이지를 열지 못했습니다. URL을 확인한 뒤 다시 시도해 주세요. |
| 로그인이 필요함 | 로그인이 필요한 게시판입니다. WebView에서 직접 로그인한 뒤 다시 시도해 주세요. |
| 접근 권한이 없음 | 이 게시글에 접근할 권한이 없습니다. 카페 가입 또는 등급을 확인해 주세요. |
| 게시글 후보를 찾지 못함 | 이 페이지에서 WOD로 보이는 게시글을 찾지 못했습니다. |
| 본문 추출 실패 | 본문을 자동으로 가져오지 못했습니다. 복사/붙여넣기 또는 수동 입력을 이용해 주세요. |
| 페이지 구조가 지원되지 않음 | 이 카페 페이지 구조는 아직 지원되지 않습니다. 수동 입력을 이용해 주세요. |
| 네트워크 연결 없음 | 네트워크 연결을 확인한 뒤 다시 시도해 주세요. |
| WebView 로딩 실패 | 페이지를 불러오지 못했습니다. 새로고침하거나 URL을 확인해 주세요. |
| 수동 입력 전환 | 수동 입력으로 전환할 수 있습니다. 가져온 내용 없이 WOD를 직접 입력합니다. |
| 홈에서 WOD 불러오기 클릭 시 URL 없음 | 등록된 카페 URL이 없습니다. 설정에서 네이버카페 URL을 먼저 등록해 주세요. |
| 등록된 카페 URL이 유효하지 않음 | 등록된 카페 URL이 유효하지 않습니다. 설정에서 URL을 확인해 주세요. |

보조 안내 문구:

- `설정한 네이버카페에서 오늘 WOD를 가져옵니다.`
- `가져온 내용은 저장 전 언제든 수정할 수 있습니다.`
- `카페 구조나 권한에 따라 가져오기가 실패할 수 있습니다.`

## 13. 보안 및 개인정보 원칙

- 앱은 네이버 ID/PW를 저장하지 않는다.
- 앱은 네이버 계정 정보를 서버로 전송하지 않는다.
- 서버가 없으므로 수집 데이터는 기기 내부에만 저장된다.
- 사용자가 직접 접근 가능한 페이지의 텍스트만 가져온다.
- 가져온 텍스트는 사용자의 WOD 기록 입력 보조 용도로만 사용한다.
- 사용자가 저장하기 전 반드시 확인하고 수정할 수 있어야 한다.
- 자동 대량 수집을 구현하지 않는다.
- 차단 회피를 구현하지 않는다.
- CAPTCHA 우회를 구현하지 않는다.
- 네이버 정책 변경 또는 카페 구조 변경 시 기능이 동작하지 않을 수 있음을 안내한다.

## 14. 권한 및 의존성 검토

- WebView로 네이버카페 페이지를 열려면 `INTERNET` 권한이 필요하다.
- v1 구현은 WebView 기반으로 시작한다.
- OkHttp/Jsoup 직접 크롤링은 v1 범위에서 제외한다.
- 외부 라이브러리 추가는 최소화한다.
- 서버 없음 원칙을 유지한다.
- 네이버 공식 API 키를 사용하지 않는다.
- OpenAI API를 사용하지 않는다.

## 16. 한계와 리스크

- 네이버카페 HTML 구조가 변경될 수 있다.
- 모바일/PC 페이지 구조가 다를 수 있다.
- iframe으로 인해 본문 추출이 어려울 수 있다.
- 로그인 상태 유지가 기기나 WebView 정책에 따라 달라질 수 있다.
- WebView 보안과 사용자 신뢰 문제가 있으므로 ID/PW 미저장 원칙을 명확히 안내해야 한다.
- 카페별 게시글 제목 형식이 다르다.
- 일부 카페에서는 자동 추출이 실패할 수 있다.
- 이 기능은 완전 자동화가 아니라 사용자 확인 기반 기능이다.
- 홈 화면 버튼은 `CafeSource` 등록 상태에 의존하므로 데이터 로딩 상태 처리가 필요하다.
